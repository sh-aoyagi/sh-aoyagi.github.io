# 自キュン 03_ブロック生成.md
# story_state.mdと直前ブロックを読んでから実行する

---

## 前提確認

以下を必ず読んでから生成を始める。

```
① story_state.md（現在地・感情値・伏線・禁止事項）
② 直前のブロック本文
③ 02_設計固定.mdの禁止事項
```

---

## STEP1：ガチャを回す

生成前に以下を決める。同じ組み合わせを3ブロック連続で使わない。

**文字数ガチャ：**
```
F01：200字（180〜220）
F02：250字（230〜270）
F03：300字（280〜320）
F04：350字（330〜370）
F05：400字（380〜420）
F06：450字（430〜470）
※同じ文字数を3ブロック連続で使わない
```

**テンポガチャ（改行数）：**
```
G01：10改行（ゆったり・詩的）
G02：15改行（標準）
G03：20改行（やや速い）
G04：25改行（速い）
G05：30改行（最速・ラリー向き）
※感情が膨らむ場面はG01〜G02
※ラリー・緊張場面はG04〜G05
```

**官能レベル：**
```
0：なし
1：触れる・距離感・体温（kyun\辞書_身体.md参照）
2：匂わせる（辞書_身体.md＋辞書_官能.mdのC・E参照）
3：Anthropicガイドライン上限（辞書_官能.md全参照）
　└ 行為を直接描写しない
　└ 直前・直後・余韻のみ描写する
　└ 「このまま朝が来なければいい」「引き返せる場所を過ぎていた」が上限
　└ 無料・有料パート問わず使用可
```

---

## STEP2：文体ガチャを1〜2回実行する

1ブロックに1〜2回、以下の確率で辞書を引いて1文を作る。

```
風景系（40%）→ kyun\辞書_風景.md から引く
身体系（40%）→ kyun\辞書_身体.md から引く
官能系（20%）→ kyun\辞書_官能.md から引く
　└ 官能レベル0の場合は身体系に振り替える

引いた辞書から2〜4要素を組み合わせて1文を作る。
この1文をブロックの中に自然に組み込む。
```

---

## STEP3：ブロックを生成する

以下のルールで生成する。

**文章ルール：**
```
・地の文では「い」を省略しない（見ていた・知っていた）
・感情は身体で表現する（嬉しかった→胸が温かくなった）
・同じ文末を3回以上続けない
・セリフの後に説明を足さない
・比喩は1場面に1つまで
・「とても」「すごく」「そして」「また」は使わない
```

**構成ルール：**
```
・テーマから逸脱しない
・story_stateの現在地を維持する
・新規要素を追加しすぎない（1ブロック1要素まで）
・禁止事項を必ず守る
```

---

## STEP3.5：ブロック終了の判断

生成が終わる前に確認する。

```
「この場面はここで切れるか？」

切れる（変化がある）→ そのまま終わる
切れない（場面が続いている）→ 延長する

延長ルール：
・文字数ガチャは目安。話の流れが優先。
・延長上限：ガチャ文字数＋100文字まで
・延長した場合は次のブロックの文字数ガチャを少なめに引く
```

話が必要としているなら伸ばす。無理に切らない。

---

## STEP4：5ブロック目の文字数チェック

5ブロック目の生成が終わったら総文字数をカウントする。

```
2000字以上 → そのまま終盤へ進む
2000字未満 → CEOに報告する

報告文：
「現在XX字です。あと1〜2ブロック追加すると
話が自然に着地できます。追加してもいいですか？」
```

**【哲学】**
無理やり着地させない。
話が必要としているなら伸ばす。
主人公の選択を見守るのが作者のスタンス。

許可が出たらブロックを追加する。
許可が出なければ現在のブロック数で終わらせる。
