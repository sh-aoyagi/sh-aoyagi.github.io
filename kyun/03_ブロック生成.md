# 自キュン 03_ブロック生成.md
# story_state.mdと直前ブロックを読んでから実行する
# ブロックを書くたびに毎回このmdを最初から実行する

---

## 前提確認

以下を必ず読んでから生成を始める。

```
① story_state.md（現在地・感情値・伏線・禁止事項・登場人物・使えない描写リスト）
② 直前のブロック本文
③ 02_設計固定.mdの禁止事項
```

---

## STEP1：ガチャを回す（毎ブロック必ず全部引き直す）

生成前に以下を全て決めて宣言する。

```
【BlockNガチャ結果】
文字数：
テンポ：
官能レベル：
辞書：
技法：
登場順序：
形容詞：
セリフ比率：
書き出し：（Block1のみ）
→ 生成開始
```

**文字数ガチャ：**
```
F01：275字（255〜295）
F02：325字（305〜345）
F03：375字（355〜395）
F04：425字（405〜445）
F05：475字（455〜495）
F06：525字（505〜545）
※同じ文字数を3ブロック連続で使わない
※2026-06-12 CEO指示で全項目+75字（5ブロック平均2000字を目標値とする）
```

**テンポガチャ（改行数）：**
```
G01：10改行（ゆったり・詩的）
G02：15改行（標準）
G03：20改行（やや速い）
G04：25改行（速い）
G05：30改行（最速・ラリー向き）
※感情が膨らむ場面はG01〜G02
※ラリー・緊張場面はG04〜G05
```

**官能レベル：**
```
0：なし
1：触れる・距離感・体温（kyun\辞書_身体.md参照）
2：匂わせる（辞書_身体.md＋辞書_官能.mdのC・E参照）
3：Anthropicガイドライン上限（辞書_官能.md全参照）
　└ 行為を直接描写しない
　└ 直前・直後・余韻のみ描写する
　└ 「このまま朝が来なければいい」「引き返せる場所を過ぎていた」が上限
　└ 無料・有料パート問わず使用可

【連続禁止ルール】
・0が2回連続したら次は必ず1以上
・3が2回連続したら次は必ず2以下
```

**辞書ガチャ：**
```
風景系（40%）→ kyun\辞書_風景.md から引く
身体系（40%）→ kyun\辞書_身体.md から引く
官能系（20%）→ kyun\辞書_官能.md から引く
　└ 官能レベル0の場合は身体系に振り替える
1ブロックに1〜2回引く
```

**技法ガチャ：**
```
T01：体言止め
T02：比喩（擬人法・直喩・隠喩・感覚転移からランダム）
T03：数字表現（時間系・回数系・距離系・金額系からランダム）
T04：多角列挙（切り口を変えて3つ・文字数不揃い）
T05：畳み掛け（同じ型を2〜3回連続）
T06：「でも」転換（1行独立）
※1ブロックに1〜2種まで
※比喩は1場面に1つまで
```

**登場順序ガチャ（物・人が登場するたびに引く）：**
```
パターンA：物→解説
パターンB：解説→物
※書き出しガチャとは別。全ブロックで適用。
※場転は新規要素ではなく必須情報。場転時は必ず場所を1行で示す。
```

**形容詞ガチャ（物・人に形容詞をつけるたびに引く）：**
```
平易系：優しそうな・くたびれた・几帳面そうな・骨張った など
難語系：泰然自若とした・寡黙で実直な・清廉潔白な など
※初登場時は必ず形容詞をつける
※2回目以降はstory_stateの記録と矛盾しないようにする
```

**セリフ比率ガチャ：**
```
高め：50%前後
標準：40%前後
低め：30%前後
```

**書き出しガチャ（Block1のみ）：**
```
S01：セリフ始まり（相手から来るセリフ推奨）
S02：道具始まり（道具＋擬人化推奨）
※情景始まりは禁止
```

## STEP2：ブロックを生成する

**固定文体ルール（毎回必ず守る）：**
```
・地の文では「い」を省略しない（見ていた・知っていた）
・感情は身体で表現する（嬉しかった→胸が温かくなった）
・感情に名前をつけない
・同じ文末を3回以上続けない
・セリフの後に説明を足さない
・セリフは1行で切る
・セリフの返しはトーンを変える（高い・真ん中・低い・生返事）
・書き出しはセリフか道具始まり（情景始まり禁止）
・「とても」「すごく」「そして」「また」は使わない
・内心は口語で短く（来ないなぁ。／あ、など）
・余白を残す・説明しすぎない・読者に委ねる
・場転時は場所を1行で示す
・店・職業・時代に矛盾する描写は禁止
　→ story_stateの「使えない描写リスト」を必ず参照する
```

**第三者の描写ルール：**
```
・初登場時のみ：肩書＋形容詞（具体1つ＋抽象1つ）
　例）店主の常盤。白髪と皺の刻まれた、泰然自若とした顔つきの男だった。
・2回目以降：肩書・形容詞は不要
```

**構成ルール：**
```
・テーマから逸脱しない
・story_stateの現在地を維持する
・新規要素を追加しすぎない（1ブロック1要素まで）
　※場転は新規要素に含まない
・禁止事項を必ず守る
```

---

## STEP3：ブロック終了の判断

生成が終わる前に確認する。

```
「この場面はここで切れるか？」

切れる（変化がある）→ そのまま終わる
切れない（場面が続いている）→ 延長する

延長ルール：
・文字数ガチャは目安。話の流れが優先。
・延長上限：ガチャ文字数＋100文字まで
・延長した場合は次のブロックの文字数ガチャを少なめに引く
```

話が必要としているなら伸ばす。無理に切らない。

---

## STEP4：5ブロック目の文字数チェック

5ブロック目の生成が終わったら総文字数をカウントする。

```
2000字以上 → そのまま終盤へ進む
2000字未満 → CEOに報告する

報告文：
「現在XX字です。あと1〜2ブロック追加すると
話が自然に着地できます。追加してもいいですか？」
```

**【哲学】**
無理やり着地させない。
話が必要としているなら伸ばす。
主人公の選択を見守るのが作者のスタンス。

許可が出たらブロックを追加する。
許可が出なければ現在のブロック数で終わらせる。
