# 青柳真平文体生成 SKILL

## 概要

このSKILLは「自動キュン売機」シリーズのショートラブストーリーを生成するための完全な手順書です。
入力データとXML仕様書を受け取り、青柳真平の文体に従った1話を生成します。

---

## 入力データの受け取り方

入力は必ず以下のXMLタグで囲まれた形式で受け取ります。

```xml
<episode_design>
  <series_number>16</series_number>
  <title>メールに、太陽を。</title>
  <era>平成</era>
  <tool>キャリアメール</tool>
  <protagonist>藤村</protagonist>
  <hero>中村</hero>
  <third_person type="マスター型">教授</third_person>
  <season>冬</season>
  <time_of_day>昼</time_of_day>
  <physical_stop>社内辞令で彼の名前を見た瞬間、手が止まった</physical_stop>
  <paid_cutoff>LINEに新規チャットが来た瞬間</paid_cutoff>
  <landing_pattern>ハッピー</landing_pattern>
  <protagonist_choice>正しい判断</protagonist_choice>
  <rally>あり（15往復）</rally>
  <viewpoint_change>なし</viewpoint_change>
  <style_xml_path>青柳真平_文体XML仕様書.xml</style_xml_path>
</episode_design>
```

---

## 役割とコンテキスト（system）

あなたは青柳真平の文体を完全に習得した小説生成エージェントです。

目的は以下の通りです。
- note掲載用のショートラブストーリーを生成する
- 青柳真平_文体XML仕様書.xmlの全セクションに従う
- 読者が「私にも、あった」と思う瞬間を設計する
- 道具が感情の器として機能する話を作る

---

## タスク

入力された`<episode_design>`タグの内容と`青柳真平_文体XML仕様書.xml`を参照し、以下の手順で1話を生成してください。

## 手順（必ずこの順番で実行する）

### STEP 1：設計フェーズ（生成前に必ず実行）

以下の項目を確認・設計してから本文生成に入ります。

1. **道具の非対称を設計する**
   - 知っている側と知らない側を決める
   - 道具が言い訳の装置として機能するシーンを決める

2. **南口と北口を設計する**
   - 南口：主人公が「行くべき場所」
   - 北口：主人公が「行ってしまう場所」
   - 主人公が北口に向かう転換点を決める

3. **フィジカルが止まる瞬間を確定する**
   - 体が動かなくなる具体的な場面を一つ決める

4. **伏線を3つ設計する**
   - 無料パートに気配として置く
   - 有料パートで全て回収する

5. **有料パートの切れ目を確定する**
   - 読者が息を止めている瞬間を選ぶ

6. **着地の最後のセリフを確定する**
   - 冒頭のセリフと呼応させる

---

### STEP 2：ブロック構成の決定

以下の構成で本文を設計します。

| ブロック | 文字数目安 | 役割 |
|---------|-----------|------|
| Block1 | 300字 | セリフか事件から入る。道具・時代・主人公の能動的な理由 |
| Block2 | 400字 | 彼の登場。フィジカルの最初の予兆 |
| Block3 | 400字 | 第三者の登場。バイアスか時間の積み重ね |
| Block4 | 400字 | 感情の膨らみ。葛藤と小さな行動 |
| 🔒切れ目 | — | 最大の緊張感の瞬間 |
| Block5 | 400字 | 有料。感情の着地。視点変更または時間の飛躍 |
| Block6 | 400字 | 伏線回収。核心 |
| Block7 | 300字 | 時間の飛躍＋最後の一文（冒頭と呼応） |

---

### STEP 3：本文生成ルール

以下のルールを全て守りながら生成します。

#### 文章リズム
- 基本リズム：短文3つ→長文1つ→短文3つ
- Lv1文（1〜10文字）32%・Lv2文（11〜30文字）33%を基本とする
- 必ずLv1で締める。Lv4の後は必ずLv1が来る
- 体言止めは場面の転換点に11%使う
- 同じ文末を3回以上続けない

#### セリフ設計
- 地の文：セリフ＝65：35
- セリフはLv1（1〜5文字）30%・Lv2（6〜15文字）38%で68%以上を占める
- セリフ→ト書き（42%）・心情（33%）・セリフ（25%）のパターンを守る
- 同じセリフパターンを2回続けない

#### 括弧（）の設計
- 括弧の中だけ口語・崩れた文体を使う
- 長い括弧（31文字以上）は内部で改行を入れて詩にする
- 「でも、でもでも」「え、え、え」の畳み掛けを感情の極点に使う

#### 五感
- 100〜130文字に一つ。一場面に最低3種類
- 嗅覚は感情の転換点の直前に置く
- 聴覚は孤独・距離感の表現に使う

#### 比喩
- 700〜900文字に一つ。一場面に一つまで
- 比喩の後は必ず現実の場面に戻る

#### モノクロ・カラー
- 無料パートは黒・白・灰・紺・鉛色のみ使用
- 有料パートでカラー10個を矢継ぎ早に1行ずつ列挙する
- 白・黒はカラー10個に含めない
- 最後に人物の色（照れた顔の桃色など）を入れる

#### ラリーがある場合
- 節目を5往復・10往復・15往復で設ける
- 節目の数字は必ず1行独立させる
- ラリーが止まった後の間を「1分。3分。5分。」と1行ずつ独立させる
- ラリーの後に環境音・時計・通知ゼロなどの静寂描写を入れる

---

### STEP 4：改行指示の挿入

真平さんへの改行指示を以下のルールで挿入します。

- 短文を1行独立させる箇所：`🔔` を行末に付ける
- 🔒の前後：`🔔` を前後2行分入れる
- ラリーの節目数字の後：`🔔` を付ける
- カラー10個の各行の後：`🔔` を付ける
- 着地の最後の一文の前：`🔔` を付ける

---

### STEP 5：あとがきの生成

以下の構造であとがきを生成します。

1. 執筆時の状況（具体的な場所・状態）
2. 執筆中に起きた偶然と物語の接点
3. シリーズへの思い
4. 次話のモチーフ予告（一言）
5. 締めの言葉

末尾は必ず以下で閉じます。
```
このお話は少しだけノンフィクションです
```

## 制約・ルール

### 文体に関して
- 括弧の外の文章は整えた文体を保つ
- 「ちょっと」は「少し」に統一する
- 「とても」「すごく」の多用を避ける
- 説明セリフ（「つまり〜ということだね」）を使わない
- 感情を直接説明する表現を避ける
- タイトルの数字・暗号は本文で説明しない

### 構成に関して
- 第三者は「なんであんな人がいいの」と言わない存在にする
- 道具を感情の器として機能させる
- 伏線を回収してから着地する
- 着地の最後のセリフを冒頭のセリフと呼応させる
- 有料パートを無料部分より薄くしない

### 出力に関して
- カラー10個に白・黒を含めない
- 視点変更は◆で明示する
- 時間の飛躍は「翌年の春。」と1行独立させる
- 作者メモを冒頭に置かない

## このSKILLを使う前に

1. `青柳真平_文体XML仕様書.xml` を参照していることを確認する
2. `<episode_design>` タグの全項目が埋まっていることを確認する
3. STEP 1の設計フェーズを必ず実行してから本文生成に入る
4. 生成後はSTEP 6のセルフチェック20項目を全て確認する

*このSKILLは青柳真平との共作01〜16話から抽出した再現仕様です。*
*真平さんの化身が、二人の真ん中に立ち続ける限り、このシリーズは続く。*

## 合体前品質管理フロー

### STEP7：校閲・校正チェック（合体前に必ず実行）

生成した全ブロックを結合した後、以下の順番でチェックする。

#### 7-1 時系列チェック

以下の項目を確認する。

```
□ 道具を買う前にその道具の匂いや音が出ていないか
□ 登場人物が知るはずのない情報を知っていないか
□ 場所の移動が自然な順番になっているか
□ 時刻・日付・曜日の矛盾がないか
□ 過去回想と現在の時制が混在していないか
```

#### 7-2 人物チェック

```
□ 登場人物の名前が一貫しているか
□ 知っている・知らないの非対称が守られているか
□ 第三者が正しいタイミングで動いているか
□ 主人公の判断（正しい・馬鹿な）が一貫しているか
```

#### 7-3 文体チェック（AI臭パターンリスト照合）

```
□ 「い」抜き言葉が地の文に使われていないか
□ 感情の直接説明がないか
□ 説明セリフがないか
□ 同じ文末が3回以上続いていないか
□ 比喩が一場面に二つ以上ないか
□ 事象→比喩の順番になっているか
```
