# 競馬記事スキル

## ⚠️ 絶対ルール
- **上から順番に実行すること。スキップ禁止。**
- **各Stepが完了したら「Step〇完了：〇〇」と出力してから次へ進む。**
- **情報が見つからない場合は再サーチする。最大3回。3回で見つからない場合のみスキップして報告する。**
## ⚠️ WebFetch・サーチ結果の検証ルール（必須）

取得した情報を使う前に以下を必ず確認する。
1つでも該当したら即破棄して再サーチする。

- 競馬場名が正しいか（東京開催なのに中山になっていないか）
- レース名がその日・その会場として成立するか
- 騎手名が実在するか（日本語として不自然な名前は即除外）
- 馬名が不自然でないか（外国語混じり・意味不明な名前は疑う）
- 同じURLで2回fetchして内容が変わる場合は信頼しない
- 推測・補完で埋めた情報は一切使わない

わからなければわからないと報告する。捏造データを含む記事は絶対に書かない。

## 「次書いて」と言われた場合

以下の順番で自動判断して記事を書く。

### Step0-A：Supabaseから今週の進捗を読む

```sql
SELECT article_type, title, status
FROM note_articles
WHERE category = '競馬'
  AND week_of = '今週の開催日（日曜）'
ORDER BY article_type ASC;
```

### Step0-B：未完成の記事を特定する

| article_type | 記事内容 | 配信タイミング |
|-------------|---------|--------------|
| 1 | 4頭の話A（2頭） | 水曜 |
| 2 | 4頭の話B（残り2頭） | 水曜 |
| 3 | メイン予想 | 木曜（枠順確定後） |
| 4 | 砂利玉7〜10R | 金曜16時以降 |
| 5 | 歴史記事 | 金曜 |

- statusが「published」以外 → 未完成
- 未完成の中でarticle_typeが最小のものを次に書く

### Step0-C：今日の日付と照合する

- 今日が水曜 → 1または2を書く
- 今日が木曜 → 3を書く（枠順確定済みか確認）
- 今日が金曜 → 4または5を書く

→ 該当する記事のStep1へ進む

## Step 1：今日の日付を確認する

- 今日の日付を取得する。
- 出力例：「Step1完了：今日は2026年6月14日（土）」

---

## Step 2：今週のメインレースを特定する

以下の優先順位で今週の重賞レースを探す。

```
G1がある → G1を選ぶ
G1がない → G2の中で賞金額が最も高いレース
G2もない → G3の中で賞金額が最も高いレース
同格が複数ある場合 → 賞金額が高い方
```

**サーチクエリのルール（日付ズレ防止）：**
- 必ず「今週 重賞レース 〇〇年〇月」の形で検索する
- 検索結果の日付を必ず確認する
- 日付が今週と合わない情報は破棄して再サーチする

- 出力例：「Step2完了：今週のメインレースは宝塚記念（G1・阪神芝2200m・6月14日）」

---

## Step 3：記事の種類を判断する

今日の日付とレース開催日を比較して、どの記事を書くか判断する。

### 記事スケジュール
| 記事 | タイミング | 内容 |
|------|-----------|------|
| ①メイン予想（注目4頭） | 木曜 | FIREスコア上位5頭＋買い目（無料）＋評価根拠（有料） |
| ②7〜10R予想（砂利玉） | 金曜16時以降 | FIREスコア解説＋10R有料 |
| ③注目4頭のうち2頭の話 | 金曜16時まで | 筆者の考え（無料・読み物） |
| ④注目4頭のうち残り2頭の話 | 金曜16時まで | 筆者の考え（無料・読み物） |
| ⑤メインレースの歴史 | 金曜 | 筆者の考え（無料・読み物） |

### 依存関係（重要）
- 記事③④⑤は記事①の情報があれば書ける（記事②を待つ必要はない）
- 記事②は独立してサーチが必要

- 出力例：「Step3完了：今日は木曜なので記事①メイン予想を書く」

## Step 4：FIREスコアを算出する

※記事①②を書く場合のみ実行。記事③④⑤はStep5へ進む。

### 4-1：レース条件の確認
以下をサーチで取得する。
- コース詳細（競馬場・距離・芝/ダート・右/左回り・直線距離・坂の有無）
- 出走馬一覧（馬名・枠番・馬番・騎手）

### 4-2：三冠馬基準の読み込み
`references/triple-crown-baseline.md` を読み込み、このレースに対応する参照三冠馬を確認する。

### 4-3：各馬のFIREスコア算出（v2・2026年6月9日改訂）

#### 【層1】確定スコア（水曜・無料公開用）68点満点

| 軸 | 配点 | 採点基準 |
|----|------|---------|
| 血統適性 | 35点 | コース・距離・馬場に合う父母父か。ディープ系・ステゴ系は東京長距離で加点 |
| 戦績 | 15点 | G1勝ち15点・G2勝ち10点・G3勝ち7点・未勝利1点。以下の補正を必ず適用する：G1初出走-3点・前走G1大敗（5着以下）-2点・海外遠征帰り初戦-1点・前走より400m以上の距離延長-2点 |
| コース実績 | 10点 | 同コース同距離で勝利経験あり10点・掲示板5点・初挑戦0点 |
| 騎手 | 5点 | トップジョッキー5点・準トップ3点・その他1点 |
| ローテーション | 3点 | 最適間隔（中4〜8週）3点・中2週以内1点・長期休養明け0点 |

**水曜時点でこの68点満点のスコアを無料パートで公開する。**

---

#### 【層2】変動係数（木〜土・有料パート用）最大32点

**枠係数（±8点）**

| 枠 | 点数 |
|----|------|
| 内枠（1〜3枠） | +8点 |
| 中枠（4〜6枠） | +4点 |
| 外枠（7〜8枠） | -4点 |
| 大外（16番以上） | -8点 |

**仕上がりタイム係数（7点）**

| タイム | 点数 |
|--------|------|
| 自己ベスト更新 | 7点 |
| 平均的なタイム | 4点 |
| 平均以下・馬なり重点 | 2点 |
| 時計未発表 | 0点 |

**仕上がりコメント係数（3点）**
※コメントは記者・関係者バイアスがかかる。絶対評価ではなく「変化の有無」で判定する。

| パターン | 判定 | 点数 |
|---------|------|------|
| 「良くなった」「シャープになった」「充実」「踏み込んだ」など前走比・先週比の**改善**を示す言葉 | 変化あり | +3点 |
| 「順調」「いい動き」「問題なし」など現状維持系 | 中立 | 0点 |
| 「リラックス」「のんびり」「感触を確かめた」など確認系 | 要注意 | -2点 |

**天気係数（2点）**

| 馬場 | 点数 |
|------|------|
| 良（晴れ・乾燥） | 2点 |
| 稍重 | 1点 |
| 重・不良 | 0点（血統により個別補正） |

---

#### 【合計】最大100点

```
確定スコア（68点）＋枠（±8）＋仕上がりT（7）＋仕上がりC（3）＋天気（2）= 最大100点
```

---

#### 有料パートの記事構成（変動係数の使い方）

有料パートでは以下の形式で各馬を解説する：

```
{馬名}：確定スコア○○点
├ 枠：「内枠なら+8で○○点→激走候補」
│    「外枠なら-8で○○点→割引き」
├ 仕上がりT：「自己ベスト更新なら+7」
└ 仕上がりC：「コメントに変化の言葉が出たら+3」

→ 最良シナリオ：○○点（激走候補）
→ 最悪シナリオ：○○点（軽視）
```

読者が枠順・追い切りを見ながら自分でスコアを更新できる設計にする。

---

**仕上がり情報が出ていない場合：**
- 「仕上がり未確認」として採点を保留する
- 確定スコア68点満点で暫定ランキングを出す
- 土曜12時以降にコメント係数・タイム係数を加算して更新する

### 4-4：スコア上位5頭を選出
- ◎本命（1位）
- ○対抗（2位）
- ▲単穴（3位）
- △要注意（4位）
- ×軽視（5位）

### 4-5：配信判断
| スコア | アクション |
|--------|-----------|
| 75点以上 | 有料note記事＋X投稿 |
| 65〜74点 | 無料X投稿のみ |
| 64点以下 | 投稿なし |

- 出力例：「Step4完了：FIREスコア算出済み。本命◎ガイアフォース82点」

## 火曜日・CEOからデータを渡された場合の動き

CEOから特別登録データが貼られたら、以下を自動実行する。

### Step A：Supabase確認
```sql
SELECT content FROM meeting_notes
WHERE category IN ('fire_score_rules','fire_score_verification')
ORDER BY created_at DESC LIMIT 5;
```

### Step B：確定スコア算出
貼られたデータからコース・距離・血統・戦績・騎手・ローテを読み取り、
FIREスコアv2指示書の【層1】68点満点で全頭採点する。

**阪神芝2200mの血統基準（宝塚記念）：**
- ステゴ系・ゴールドシップ系・ハーツクライ系 → 高評価（スタミナ×パワー）
- ロベルト系・エピファネイア系 → 高評価（阪神内回り適性）
- 軽い芝向きマイラー血統 → 減点

**東京芝2400mの血統基準（ダービー・オークス）：**
- ディープ系・ステゴ系×東京実績 → 高評価
- キタサン系・ハーツ系 → 中評価
- スプリント特化型 → 減点

**東京芝1600mの血統基準（安田記念）：**
- キンカメ系・ロベルト系・Storm Bird系 → 高評価
- ディープ系でもマイル実績あり → 中評価
- スタミナ特化型（菊花賞・天皇賞春血統）→ 減点

### Step C：記事生成
上位4頭を選出し、以下のフォーマットで無料パート記事を生成する。

```
{前週との接続・挨拶1〜2行}

私たちが注目する4頭はこれだ。

━━━━━━━━━━━━━━━━━━━━
◎ 1位：{馬名A}
○ 2位：＊＊＊＊＊＊＊
○ 3位：＊＊＊＊＊＊＊
△ 4位：{馬名D}
━━━━━━━━━━━━━━━━━━━━

{馬名Aの説明・1〜2行・事実のみ}
{馬名Dの説明・「なぜ4位？」を匂わせる・1〜2行}

真ん中の2頭と買い目は有料パートへ（¥100）
```

**隠し方のルール：**
- 1位：前走実績が明確な馬 → 見せる
- 2位・3位 → 隠す
- 4位：誰でも知っている人気馬 → 見せる（「なぜ4位？」の動機）

### Step D：Supabase保存
```sql
INSERT INTO note_articles
(category, article_type, title, free_content, status, race_name, week_of)
VALUES
('競馬', 3, '{レース名}予想｜FIREスコアが示す今週の4頭',
 '{生成した無料パート記事}', 'draft', '{レース名}', '{開催日}');
```

### Step E：報告
「確定スコア算出完了。上位4頭と無料パート記事をSupabaseに保存しました。」
と報告する。
