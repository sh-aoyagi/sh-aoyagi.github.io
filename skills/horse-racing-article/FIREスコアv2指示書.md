## なぜ変えたか（背景）

2026年5月のダービー・オークスの結果データとFIREスコアを照合した結果、以下の問題が判明した。

1. **「枠」が過大評価だった**
   ロブチェン（8枠17番）はFIREスコアでは枠の減点が大きく低評価になったが実際1着。
   枠単体の影響は最大±8点に抑えるべきだった。

2. **「コース実績」が戦績に埋もれていた**
   ゴーイントゥスカイ（青葉賞勝ち・東京2400m実績あり）を正確に評価するには、
   戦績とは別に「同コース同距離で勝ったことがあるか」を独立した変数にする必要があった。

3. **「コメント係数」の方向性が逆だった**
   関係者コメントはポジティブバイアスがかかるため、絶対評価できない。
   正しいアプローチは「前走比・先週比の変化を示す言葉があるか」のパターン認識。

4. **スコアが静的すぎた**
   水曜に出したスコアが土曜まで変わらず、枠・仕上がりの情報が反映できなかった。
   3層構造（確定スコア＋変動係数）にして記事フローと一致させる。

5. **「G1初出走ペナルティ」が存在しなかった**（安田記念2026検証より判明）
   トロヴァトーレ（G2連勝でG1初挑戦・最外枠）を算出では57点2位に評価したが実際9着。
   実際の記事では「G1初出走×最外枠は条件が重なりすぎる」として×軽視62点と正しく評価していた。
   戦績点に「G1初出走-3点・前走G1大敗-2点・海外帰り初戦-1点」の補正を追加することで解消。

## 新FIREスコア設計（v2）

### 【層1】確定スコア　68点満点
**水曜・無料パートで公開する数字**

| 軸 | 配点 | 採点基準 |
|----|------|---------|
| 血統適性 | 35点 | コース・距離・馬場に合う父母父か |
| 戦績 | 15点 | G1勝ち15点・G2勝ち10点・G3勝ち7点・未勝利1点。以下の補正を必ず適用する：G1初出走-3点・前走G1大敗（5着以下）-2点・海外遠征帰り初戦-1点・前走より400m以上の距離延長-2点 |
| コース実績 | 10点 | 同コース同距離での勝利10点・掲示板5点・初挑戦0点 |
| 騎手 | 5点 | トップジョッキー5点・準トップ3点・その他1点 |
| ローテーション | 3点 | 中4〜8週3点・中2週以内1点・長期休養明け0点 |

---

### 【層2】変動係数　最大+32点
**木〜土・有料パートで解説する数字**

**枠係数（±8点）**
| 枠 | 点数 |
|----|------|
| 内枠（1〜3枠） | +8点 |
| 中枠（4〜6枠） | +4点 |
| 外枠（7〜8枠） | -4点 |
| 大外（16番以上） | -8点 |

**仕上がりタイム係数（7点）**
| タイム | 点数 |
|--------|------|
| 自己ベスト更新 | 7点 |
| 平均的なタイム | 4点 |
| 平均以下・馬なり重点 | 2点 |
| 時計未発表 | 0点 |

**仕上がりコメント係数（3点）**
※コメントは関係者バイアスがかかる。絶対評価禁止。「変化の有無」だけ見る。

| パターン | 点数 |
|---------|------|
| 「良くなった」「シャープになった」「充実」「踏み込んだ」など前走比の改善を示す言葉 | +3点 |
| 「順調」「いい動き」「問題なし」など現状維持系 | 0点 |
| 「リラックス」「のんびり」「感触を確かめた」など確認・状態チェック系 | -2点 |

**天気係数（2点）**
| 馬場 | 点数 |
|------|------|
| 良（晴れ） | 2点 |
| 稍重 | 1点 |
| 重・不良 | 0点 |

---

### 【合計】最大100点
```
確定スコア（68）＋枠（±8）＋仕上がりT（7）＋仕上がりC（3）＋天気（2）= 最大100点
```

## 記事フローへの反映

### 水曜（無料パート）
- 確定スコア68点満点のランキングを出す
- 「この馬は枠と仕上がり次第で浮く/沈む」とヒントだけ出す
- 具体的な点数は出さない

### 木〜土（有料パート）
各馬を以下の形式で解説する：

```
{馬名}：確定スコア○○点

枠：内枠なら+8で○○点→激走候補
　　外枠なら-8で○○点→割引き

仕上がりT：自己ベスト更新なら+7点
仕上がりC：コメントに変化の言葉が出たら+3点

→ 最良シナリオ：○○点（激走候補）
→ 最悪シナリオ：○○点（軽視）
```

読者が自分で枠順・追い切りを確認しながらスコアを更新できる設計にする。
これがFIREスコアを「思考ツール」として使ってもらうための仕掛け。

## 無料パートの記事フォーマット（v2）

### 基本構造

```
{前週との接続・挨拶1〜2行}

FIREスコア（Final Insight Racing Evaluation）は、血統・戦績・コース実績・
騎手・ローテーションの5軸で馬を採点する独自指標。
ジンクスも新聞評価も入れない。馬が証明した事実だけを見る。

今週、スコアを出した。

私たちが注目する4頭はこれだ。

━━━━━━━━━━━━━━━━━━━━
◎ 1位：{馬名A}
○ 2位：＊＊＊＊＊＊＊
○ 3位：＊＊＊＊＊＊＊
△ 4位：{馬名D}
━━━━━━━━━━━━━━━━━━━━

{馬名A}の説明（1〜2行・事実のみ）
{馬名D}の説明（1〜2行・「なぜ4位？」を匂わせる）
真ん中2頭は有料パートで。

詳細・買い目は有料パートへ（¥100）
```

### 隠し方のルール

- 1位：下馬評上位or前走実績が明確な馬 → 見せる（読者の安心感）
- 2位・3位：隠す（「誰だ？」の動機）
- 4位：誰でも知っている人気馬 → 見せる（「なぜ4位？」の動機）

### 隠し方が機能するパターン

```
パターンA（人気馬が4位）
◎ 1位：＊＊＊＊（無名・高スコア）
○ 2位：＊＊＊＊
○ 3位：人気馬B
△ 4位：1番人気A  ← 「え、なんで4位？」

パターンB（波乱含み）
◎ 1位：人気馬A  ← 安心感
○ 2位：＊＊＊＊
○ 3位：＊＊＊＊
△ 4位：人気馬B  ← 「真ん中2頭は穴馬？」

パターンC（完全波乱）
◎ 1位：＊＊＊＊
○ 2位：人気馬A  ← 1位じゃない！
○ 3位：＊＊＊＊
△ 4位：人気馬B  ← 「1位は誰だ？」
```
