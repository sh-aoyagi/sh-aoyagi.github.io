## 有料部分フォーマット（¥100）

### Step1：書き出しパターンを選ぶ

枠順が確定した瞬間の筆者の感情で書き出しを選ぶ。

---

**パターンA：「うわー最悪」**
本命・対抗が不利な枠（大外・内枠すぎ）を引いた時。

```
{本命馬名}、{枠}枠{馬番}番。
うわー、最悪。
正直そう思った。

{ジンクスの話：「大外から1人気は消え」みたいな話が
 今週末SNSに流れるのは目に見えてる}

でもFIREスコアはジンクスを計算に入れない。
馬は人語を解さない。枠番を見て気にするのは人間だけだ。
バイアスは排除する。変わらない要素だけを見る。
```

---

**パターンB：「きたー！これは買った！」**
本命が理想の枠を引いた時。

```
{本命馬名}、{枠}枠{馬番}番。
きたー。これは買った。

{なぜその枠が理想なのかの説明・1〜2行}

FIREスコアが示していた通りの展開になった。
変わらない要素が、変わる要素（枠）を味方につけた。
```

---

**パターンC：「微妙だなー、検討しなおさなきゃ」**
枠でスコアが動いて順位が入れ替わった時。

```
{本命馬名}、{枠}枠{馬番}番。
微妙だなー。正直、検討しなおした。

枠が入って、スコアが動いた。
{暫定スコア}点だった{馬名}が{最終スコア}点になった。
代わりに{別の馬名}が浮上してきた。

これがFIREスコアの正直なところだ。
```

---

### Step2：1〜4位の評価根拠を書く

各馬について以下の順番で書く：

```
【{◎○▲△}　{馬名}】

{血統とこのレースの相性：1〜2文}
{過去戦績から見えること：1〜2文}
{枠・騎手を加えた最終評価：1文}
FIREスコア：{点数}点
```

---

### Step3：5枠目のジャッジ（クライマックス）

有料パートの核心。「買う」か「消す」かの判断を明示する。

```
そして5枠目、{馬名}について。

{枠確定前の暫定スコア}点だったこの馬、
枠が{枠番}番に入って{最終スコア}点になった。

【買いましょう】
→ 理由：{枠・仕上がりでプラスになった根拠1〜2文}
　 この馬をワイドの3頭目に加える。

または

【買うのをやめましょう】
→ 理由：{枠・仕上がりでマイナスになった根拠1〜2文}
　 この馬は消し。3連複の軸から外す。
```

---

### Step4：まとめ・買い目の根拠


## スコア計算ルール（記事作成時の手順）

### 木曜時点（枠確定後）のスコア算出順序

```
① 血統（40点）：triple-crown-baseline.mdの基準で採点
② 戦績（20点）：過去成績・距離実績・コース実績で採点
③ 騎手（5点）：コース・距離との相性で採点
④ 枠（15点）：枠確定後に採点
　 → ここでスコアが±5〜10点動く
　 → 70点前後の馬の順位が入れ替わる可能性あり
⑤ 仕上がり（18点）：土曜12時以降に採点・追記
⑥ 天気（2点）：当日朝に採点・追記
```

### 暫定スコア（水曜・記事③④用）
血統40＋戦績20＋騎手5＝**65点満点**で算出

### 最終スコア（木曜・記事①用）
全6軸合計＝**100点満点**で算出
