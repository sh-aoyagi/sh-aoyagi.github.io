# サーチルール（日付ズレ・情報取得不可を防ぐ）

## ⚠️ 大前提：情報が出るタイミングがある

競馬の情報は「いつでも取れる」わけではない。
タイミングより前にサーチしても見つからないのは当然。
「見つからない＝存在しない」ではなく「まだ出ていない」。

---

## 情報が出るタイミング一覧

| 情報の種類 | 公開タイミング |
|-----------|--------------|
| メインレース出走馬 | 火曜〜水曜（枠順確定） |
| 7〜10R出走表 | 木曜〜金曜 |
| 騎手確定 | 木曜〜金曜 |
| 追い切り情報（仕上がり） | 土曜12時以降 |
| オッズ | 土曜〜日曜（発売開始後） |
| 天気予報 | 土曜朝 |

**7〜10Rの出走表は木曜〜金曜に出る。それより前にサーチしても見つからない。**

---

## 日付確認ルール（最重要）

### Step1：必ず今日の日付を取得してから検索する

```
✅ 正しい手順
1. 今日の日付を確認する（例：2026年6月11日木曜）
2. 開催日を計算する（今週の日曜＝6月14日）
3. クエリに年月日を必ず含める

❌ やってはいけないこと
・日付を確認せずにサーチする
・「今週」「最新」だけで検索する（古い情報が出る）
・検索結果の日付を確認しないまま使う
```

### Step2：検索結果の日付を必ず確認する

```
✅ 使っていい情報：今週の開催日に一致しているもの
❌ 捨てる情報：日付が違うもの・「来週」「先週」のもの

日付が合わない場合 → 再サーチする（最大3回）
3回やっても合わない場合 → 「日付不一致のため情報なし」と報告する
```

## レース別・正しいサーチクエリ

### メインレース（G1/G2/G3）
```
✅ 「{レース名} {年} 出走馬 枠順」
✅ 「{レース名} 2026 出走表」
✅ 「{レース名} {月}{日} 出走馬」

例）「宝塚記念 2026 出走馬 枠順」
例）「宝塚記念 2026年6月14日 出走表」
```

### 7〜10R（条件戦・砂利玉レース）
条件戦はレース名がない場合が多い。以下のクエリで取得する。

```
✅ 第一優先：「{競馬場} {月}{日} {R番号} 出走表」
例）「阪神 6月14日 7レース 出走表 2026」
例）「阪神 6月14日 8R 出走馬 2026」

✅ 第二優先：netkeiba の出走表ページをfetch
例）「netkeiba 阪神 2026年6月14日 7R 出走」

✅ 第三優先：「{競馬場} {月}{日} 全レース 出走表」
例）「阪神競馬 6月14日 全レース出走表 2026」
```

### 追い切り情報（仕上がり）
```
✅ 「{レース名} 追い切り {年}{月}」
✅ 「{馬名} 最終追い切り {年}」

例）「宝塚記念 追い切り 2026年6月」
例）「クロワデュノール 最終追い切り 2026」

⚠️ 土曜12時より前はまだ出ていない場合が多い。
   見つからない場合は「仕上がり未確認・土曜12時以降に更新」として保留する。
```

### 天気
```
✅ 「{競馬場所在地} 天気 {月}{日}」
例）「宝塚市 天気 6月14日」
例）「阪神競馬場 天気予報 6月14日 2026」
```

## 「出走馬データ取得不可」の禁止ルール

### ❌ 絶対にやってはいけないこと
- 「出走馬データ取得不可のため本命印なし」として記事を出す
- 出走馬不明のままコース解説だけ書く
- 3回サーチしないうちに諦める

### ✅ やるべきこと
```
1回目サーチで見つからない
↓
クエリを変えて2回目サーチ
↓
まだ見つからない
↓
クエリを変えて3回目サーチ
↓
それでも見つからない
↓
「{競馬場}{R番号}の出走表は現時点（{日時}）では未公開です。
 公開タイミング（木〜金曜）に再度サーチしてください。」
と報告して記事作成を中断する。
```

## サーチ結果の信頼性チェック

取得した情報を使う前に以下を確認する：

```
□ 日付は今週の開催日と一致しているか？
□ 競馬場は正しいか？（東京/阪神/中山/京都/函館など）
□ レース番号は正しいか？
□ 情報源は信頼できるか？（JRA公式・netkeiba・スポーツナビ優先）
```

信頼できる情報源の優先順位：
1. JRA公式（jra.go.jp）
2. netkeiba（race.netkeiba.com）
3. 競馬ラボ（keibalab.jp）← 7〜10R出走表はここが最も確実
4. スポーツナビ競馬・ウマニティ

## 7〜10R出走表の取得方法

### ⚠️ keibalab.jpはbot検出でfetch不可
URLパターンは判明しているが直接fetchできない。
参考情報として記録のみ：
```
https://www.keibalab.jp/db/race/{YYYYMMDD}{場コード2桁}{R番号2桁}/
場コード：01札幌 02函館 03福島 04新潟 05東京
　　　　　06中山 07中京 08京都 09阪神 10小倉
```

### ✅ 実際の取得手順（netkeiba・JRA公式経由）

4レース分を個別に検索して取得する。

```
検索クエリのパターン：
「{競馬場名} {年月日} {R番号}レース 出走馬」

例（6/14阪神の場合）：
「阪神競馬 2026年6月14日 7レース 出走馬」
「阪神競馬 2026年6月14日 8レース 出走馬」
「阪神競馬 2026年6月14日 9レース 出走馬」
「阪神競馬 2026年6月14日 10レース 出走馬」
```

取得できない場合はJRA公式も試す：
```
https://www.jra.go.jp/keiba/calendar2026/2026/{月}/{月日}.html
```

### 取得できない場合のルール
- 3回サーチしても取得できない → 「{R番号}R：出走馬未確認（木〜金曜に再取得）」と明記
- 「出走馬不明のため本命印なし」として記事を書くことは禁止
- 情報が出るタイミング（木曜16時以降）を過ぎても取得できない場合のみ報告する
