
## A：メイン予想記事（記事③・水曜・有料¥100）

`references\article-format-main.md` を読んでから書く。

### 無料パート（データ先行型・物語型禁止）

```

今週は{レース名}です。早速行きましょう！

━━━━━━━━━━━━━━━━━━━━
 ■ FIREスコア 上位5頭（暫定版・枠確定前）
━━━━━━━━━━━━━━━━━━━━

◎ 本命：【1位】{馬名}　{枠}枠{馬番}番　{騎手}
○ 対抗：【2位】{馬名}　{枠}枠{馬番}番　{騎手}
▲ 単穴：【3位】{馬名}　{枠}枠{馬番}番　{騎手}

■ 買い目（暫定版）

各馬の評価根拠は、有料パートで！
```

### 有料パート（青柳文体・物語型）

`references\aoyagi-style.md` を読んでから書く。

**書き出しパターンのローテーションルール：**

Supabaseのnote_articlesテーブルから直近の記事を確認して、
使用済みのパターン番号を避けて次のパターンを選ぶ。
同じパターンを連続して使わない。

```
パターンA（枠が最悪の場合）：
A1「引き、悪すぎだろぉぉぉおおおぉぉぉ...」
A2「うそだろ！？またかよ...」
A3「そうです、きっと私が悪いんです...」
A4「代表して謝っておきます。引きが悪くてごめんなさい...」
A5「今年は年男なんすけど、これはちょっとキツい...」

パターンB（枠が最高の場合）：
B1「ありがとうございます、ありがとうございます...！！」
B2「神様ってちゃーんと見てるんっすよ、ええ。」
B3「単勝買っても500円にならないレースがだーいすき！」
B4「ついに伝説の目撃者になれますね！」
B5「これはもう、買わない理由を探す方が難しい。」

パターンC（枠が微妙・判断が難しい場合）：
C1「ぬぬぬぬぬぬぬぬぬ......　どーします？」
C2「予想記事書いててなんですが、カンでいきたくなる瞬間ですよ。」
C3「ちょっとググってくるので待っててもらっていいですか？　あ、無理ですか。」
C4「ちょいちょいある、皆さんの意見が欲しくなるレースって感じですね。」
C5「スコアは出てる。でも手が止まる。そういう週ってあるんですよ。」
```

**構成：**
```
① 書き出し（A・B・Cのいずれか1つ・ローテーション）
② 1〜4位の評価根拠（各馬・血統×コース適性）
③ 5枠目のジャッジ（買う・消す）← クライマックス
④ まとめ・買い目の根拠
```

**ペルソナ10人チェック（必須）：**
`references\aoyagi-style.md` のペルソナ評価フローを実行する。
6人以上「購入する」→ 採用
5人以下 → 修正して再チェック（最大3回）

## 記事完成後の確認

```
□ 無料パートはデータ先行型になっているか（物語型になっていないか）
□ 有料パートは青柳文体になっているか
□ FIREスコアの数値が正式配点（68点＋32点）になっているか
□ ペルソナ10人チェックを実施したか（記事③のみ）
□ 砂利玉の2頭目が＊＊＊＊で隠されているか
□ 書き出しパターンが前回と被っていないか
□ 買い目が入っているか
□ 免責事項が入っているか
```

```
的中・勝馬を保証するものではありません。
馬券の購入は必ずご自身の判断と責任のもとで行ってください。
```
