# 競馬 04_校正.md
# 03_品質チェックの結果を受けて実行する

---

## 絶対ルール

```
内容変更禁止
削除禁止
追加禁止
以下の4種類のみ修正可能：
　① てにをは
　② 誤字
　③ 重複表現
　④ 連続パターンの解消
```

---

## 修正①：てにをは

助詞の誤りを修正する。内容は変えない。

---

## 修正②：誤字

明らかな誤字のみ修正する。
表記ゆれ（ひらがな・カタカナ）は原則触らない。

---

## 修正③：重複表現

同じ言葉が近い場所に重複している場合のみ修正する。
例：「枠を引いて、枠の結果が」→「枠を引いて、その結果が」

---

## 修正④：連続パターンの解消

以下のパターンが3回以上続いている場合のみ修正する。

**文末の連続：**
```
～た。～た。～た。が続く場合：
3つ目の文末を以下のどれかに変える
・体言止め（名詞で終わる）
・「～る」現在形
・「～だ」
・短文1語

例）
修正前：枠を確認した。スコアが動いた。手が止まった。
修正後：枠を確認した。スコアが動いた。手が、止まる。
```

**描写の連続（多角列挙への変換）：**
```
情景・情景・情景が続く場合：
3つを多角列挙に変換する

切り口を別々にする：
・色・擬人化・感情の距離・時間・温度・音など
・文字数を揃えない（短・中・長 または 中・短・長）
・同じ文字数を2つ以上続けない
```

---

## 品質チェックで指摘された問題への対応

03_品質チェックで「問題あり」が出た場合：

```
⑩ペルソナチェック5人以下の場合：
　→ 無料パートの末尾「引き」を強化する
　→ 「変化の言葉」「なんで？を引き出す落差」を追加する
　→ 再チェックを03_品質チェックで実行する

⑥物語型になっている場合：
　→ 無料パートをデータ先行型に書き直す
　→ 物語型の文章は有料パートに移動する

⑦AI臭がある場合：
　→ references\aoyagi-style.mdの禁止表現リストと照合する
　→ 該当箇所を青柳文体に書き直す
```

## 最終出力

校正完了後、記事の完成版を出力する。

```
出力形式：
・無料パートと有料パートを区切り線で分けて出力
・タイトルを先頭に入れる
・免責事項を末尾に入れる
・Supabaseのnote_articlesに保存する
　（category='競馬'・week_of=今週の開催日・article_type=該当番号・status='draft'）
```

---

## 注意

修正は最小限にとどめる。
迷ったら修正しない。
「直した方がいい気がする」は修正理由にならない。
内容・構成・スコアの数値は絶対に変えない。
