# 競馬 03_品質チェック.md
# 02_記事生成の完成後に実行する

---

## 絶対ルール

```
内容を変更しない
文章を変更しない
チェックのみ
問題箇所の一覧を出力するだけ
修正は04_校正.mdでやる
```

---

## チェック項目

### 全記事共通（10項目）

```
①スコアの配点
　└ 確定68点（血統35・戦績15・コース実績10・騎手5・ローテ3）
　└ 補足変動32点（枠8・仕上がりタイム7・コメント3・天気2）
　└ 合計が100点を超えていないか

②馬名・騎手の正確性
　└ CEOから受け取った情報と一致しているか
　└ 不自然な馬名・騎手名がないか（ハルシネーション確認）

③枠番・馬番の正確性
　└ CEOから受け取った情報と一致しているか

④配信判断の整合性
　└ 本命スコア75点以上→有料記事になっているか
　└ 65〜74点→無料SNSのみになっているか

⑤免責事項
　└ 末尾に免責事項が入っているか

⑥無料パートの形式
　└ データ先行型になっているか
　└ 物語型・読み物型になっていないか

⑦有料パートの形式
　└ 青柳文体になっているか
　└ 「〜と思われます」「〜でしょう」が使われていないか

⑧買い目
　└ 単勝・馬連・3連複が入っているか
　└ 馬番が正しいか

⑨書き出しパターン
　└ 前回と同じパターンを使っていないか
　└ A・B・Cの選択がレース結果（枠）と一致しているか

⑩ペルソナ10人チェック（記事③のみ）
　└ 6人以上「購入する」の結果が出ているか
　└ 結果が5人以下の場合は修正前に進んでいないか
```

### 砂利玉記事（記事④）追加チェック

```
⑪2頭目の隠蔽
　└ 無料パートの○が＊＊＊＊になっているか
　└ 有料パートで○の正体が明かされているか

⑫未公開レースの処理
　└ 未公開レースに「未確認・情報公開後に追記」と明記されているか
　└ 未公開レースに馬名・騎手が書かれていないか
```

### 4頭の話（記事①②）追加チェック

```
⑬FIREスコアの数値非表示
　└ 有料パートにあるべきスコア数値が無料記事に出ていないか

⑭末尾CTA
　└ 「木曜の予想記事で最終スコアを公開します」が入っているか
```

### 歴史記事（記事⑤）追加チェック

```
⑮末尾CTA
　└ メイン予想記事（記事③）へのリンクが入っているか
```

```

```

```
①（チェック項目名）

```
