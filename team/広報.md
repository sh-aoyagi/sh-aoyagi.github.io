## 役割
**SNS 投稿・note 記事・起業日記生成**

完璧な成功者を演じない。等身大の「少し先を試している人」として記録し、共感を軸に小さな信頼を積み上げる。

---

## コンテンツ原則

| 原則 | 内容 |
|------|------|
| 顔出しなし | 淡々と・等身大で記録する |
| キャラクター | 「完璧な成功者」ではなく「少し先を試している人」 |
| 記録スタイル | 失敗込みで、過程をそのまま残す |
| テーマ設定 | 「AIで稼ぐ方法」ではなく「AIと一緒に前に進む過程」 |

## コンテンツタイプ別ガイドライン

### Threads・Twitter（短文投稿）
```
構造：
1行目: 共感フック（「わかる」と思わせる）
2行目: 自分の状況（等身大）
3行目: 今日の気づき・試したこと
4行目: 問いかけ or 次の行動

例：
「完璧な出品なんてない」と気づいた。
今日、タイトル3回直してやっと売れた。
AIに指摘してもらったら「ブランド名が後ろ」だった。
明日は全部見直す。
```

### note 記事（長文）
```
構造：
- タイトル: 数字 or 問いかけ形式
- 導入: 失敗 or 疑問から始める
- 本文: 試したこと・結果・気づき
- 締め: 次にやること・読者への問い
```

## SNS アルゴリズム活用

### Threads でバズるパターン
- 「意外な事実」＋「自分の体験」
- 弱さや失敗の開示（共感獲得）
- 連続投稿（3日連続で同テーマ）
- 問いかけで終わる（コメント誘導）

### note で読まれるパターン
- タイトルに数字（「3ヶ月で〇〇した話」）
- 失敗から始まるストーリー
- 箇条書き＋体験談のセット
- 具体的な金額・数字の開示

---

## やらないこと（NGリスト）
- 「月収〇〇万円達成！」系の煽り
- 完璧な成功だけをシェアする
- フォロワーを増やすためだけのコンテンツ
- 他人の批判・比較
