## 2. 無料記事の設計原則（2026-05-28 確定）

```
【記事役割分離】
売り物：予想記事のみ
その他（木・金・はじめて）：アカウント育成＋私への文体・思考法インプット＋アフィリ布石

【タイトルフォーマット】
レース名＋年号｜馬名Aと馬名B、短い読ませるフレーズ
例：「日本ダービー2026｜ロブチェンとリアライズシリウス、証明か復讐か"世代最強の第3ラウンド"」
フレーズは対比・引用符・動詞を含む。事実として成立していること。

【三役割テンプレート（木曜・本命系記事の構造）】
リベンジ = 前走で悔しい思いをした馬（→ 感情的共感）
証明     = 人気・実力No.1（→ 読者との一体感）
破壊     = FIREスコア本命（→ 名前を出さず「気配」のみ。有料記事へ引き込む）

【FIREスコアの哲学（絶対に守る）】
ジンクス = 「下馬評で勝つわけない」という人間のバイアス
馬は人語を解さない。人間の作ったバイアスはスコアに入れない。
変わらない要素（固定）：血脈 × コース仕様
変わる要素（更新）：馬が作った実績データ・3ハロンタイム
除外する要素：ジンクス・新聞評価・気配（人間由来の主観）

ジンクスは記事の中では「緊張感の演出素材」として使う。
リスク要因として書かない（FIREの哲学と矛盾するため）。

【ウマ娘隠喩の方針（将来のアフィリ布石）】
ライスシャワー = 空気を読まずに勝つ → 下馬評を覆す馬・騎手の隠喩として使う
サトノの血     = ジンクスの象徴 → ジンクスが破られた例として使う
直接ゲーム名・キャラ名は有料記事・収益記事に出さない（リーガルリスク）
「この筆者はウマ娘好きだな」と読者に思わせる程度の隠喩で留める
アフィリが取れたタイミングで接続する
```
